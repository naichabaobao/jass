return

return 用于结束当前函数执行，并返回值（或直接返回）。

基础语法

// returns nothing
return

// returns integer
return 0

常见用法：提前退出（Guard Clause）

当参数不合法、状态不满足时，推荐在函数前部直接 return，减少嵌套层级。

function SafeKill takes unit u returns nothing
    if u == null then
        return
    endif
    if GetWidgetLife(u) <= 0.405 then
        return
    endif
    call KillUnit(u)
endfunction

Return Bug 教程（War3 旧版本历史问题）

背景：在较老环境中，某些“并非所有路径都显式 return”的写法可能被接受，形成所谓 return bug 相关历史习惯。
这类写法在不同工具链/版本下行为不一致，容易造成隐藏问题。

不推荐写法（依赖历史行为）

function Foo takes boolean ok returns integer
    if ok then
        return 1
    endif
    // 这里没有明确 return，行为依赖环境
endfunction

推荐写法（稳定、可移植）

无论目标版本如何，始终保证所有路径显式返回，避免依赖历史兼容行为。 
function Foo takes boolean ok returns integer
    if ok then
        return 1
    endif
    return 0
endfunction

实践建议

  • returns T 的函数：保证所有路径都 return T。
  • returns nothing：可在任意位置用 return 提前结束。
  • 不要把 return 放在难以触达的位置，保持分支清晰可读。
