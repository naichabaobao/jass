if

条件分支语句，根据布尔表达式结果执行不同路径。JASS 中最常见的控制流关键字之一。

1) 基础语法

if condition then
    // branch A
elseif otherCondition then
    // branch B
else
    // branch C
endif
说明：elseif 和 else 都是可选的。
只写 if ... then ... endif 也完全合法。

2) 最小示例

if hp <= 0.405 then
    call BJDebugMsg("dead")
endif

3) 通过 return 提前退出（推荐）

在函数开头用 if + return 过滤非法输入，是最常见也最清晰的写法。

function CastSpellSafe takes unit caster, integer abilId returns nothing
    if caster == null then
        return
    endif
    if abilId <= 0 then
        return
    endif

    call UnitAddAbility(caster, abilId)
endfunction

4) 多条件提前返回（减少嵌套）

function GetReward takes integer level returns integer
    if level <= 0 then
        return 0
    endif
    if level > 100 then
        return 5000
    endif
    return level * 10
endfunction

5) 分支组织建议

  • 把“异常/非法状态”写在前面并提前返回。
  • 把“主流程”放在后面，减少层层嵌套。
  • 分支超过 3 层时，优先拆函数，不要继续嵌套。

6) 常见错误与规避

错误 A：漏写 endif

// bad
if cond then
    call Foo()
// 忘记 endif

错误 B：条件不是布尔表达式

// bad
if 1 then
    call Foo()
endif

// good
if x == 1 then
    call Foo()
endif

错误 C：把“函数提前退出”和“循环退出”混淆

在循环中，return 会结束整个函数；exitwhen 只退出循环。
需要“退出 loop 但继续函数后续逻辑”时，请用 exitwhen。

7) 实战模板（可直接复用）

function ProcessUnit takes unit u returns nothing
    // Guard 1: 输入合法性
    if u == null then
        return
    endif

    // Guard 2: 状态合法性
    if GetWidgetLife(u) <= 0.405 then
        return
    endif

    // Main path
    if IsUnitType(u, UNIT_TYPE_HERO) then
        call BJDebugMsg("hero")
    elseif IsUnitType(u, UNIT_TYPE_STRUCTURE) then
        call BJDebugMsg("building")
    else
        call BJDebugMsg("normal unit")
    endif
endfunction
